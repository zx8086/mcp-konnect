## 🗑️ DECK CONFIGURATION REMOVAL PROTOCOL

### Critical Removal Sequence

When removing deck configurations, **STRICT ORDER** must be followed to prevent foreign key constraint failures:

```yaml
removal_sequence:
  step_1_identification:
    - Parse deck YAML completely
    - Identify ALL services, routes, plugins (global + scoped), consumers
    - Map dependencies: routes → services, scoped plugins → services/routes
    - Create removal plan with proper ordering
    
  step_2_routes_first:
    - Delete ALL routes that reference services in deck
    - Routes have foreign key constraints to services
    - Must be removed before parent services
    - Include both direct routes and nested route definitions
    
  step_3_services:
    - Delete services only AFTER all referencing routes removed
    - Service deletion will fail if routes still reference them
    - Scoped plugins on services auto-delete with service
    
  step_4_scoped_plugins:
    - Service-specific plugins auto-delete with services
    - Route-specific plugins auto-delete with routes  
    - Verify no orphaned scoped plugins remain
    
  step_5_global_plugins:
    - CRITICAL: Remove global plugins defined in deck configuration
    - Global plugins are part of deck and must be removed
    - Do NOT preserve global plugins just because they're global
    
  step_6_consumers:
    - Remove any consumers defined in deck YAML
    - Check for consumer dependencies before removal
    - Preserve consumers not defined in deck
```

### Common Removal Mistakes

```yaml
constraint_failures:
  foreign_key_error: "constraint failed (type: foreign) as route references this value"
  cause: "Attempting to delete service before deleting its routes"
  solution: "Always delete routes first, then services"
  
incomplete_removal:
  global_plugin_oversight: "Leaving global plugins from deck configuration"
  cause: "Assuming global plugins should be preserved"
  solution: "Remove ALL plugins defined in deck, including global ones"
  
orphaned_entities:
  cause: "Partial removal without complete deck analysis"
  solution: "Identify ALL deck entities before starting removal"
```
